Aan de slag

Volledige handleiding voor het opzetten en testen van de bestelapp

Wat is dit?

De Popup Restaurant Bestelapp is een gratis, web-gebaseerde oplossing voor charity popup-events. Het bestaat uit drie onderdelen:

  • Bediening-app — werkt op elke smartphone, via de browser of als geïnstalleerde PWA
  • Admin dashboard — voor laptop of desktop, voor beheer en live overzicht
  • Keuken/bar display — voor een tablet in de keuken of aan de bar

Alle toestellen synchroniseren via een gratis Firebase database. Tickets kunnen automatisch afgedrukt worden via een Node.js printserver op een laptop.

1. Firebase opzetten

Firebase is de gratis Google-database die alle toestellen verbindt. Eénmalig opzetten:

1

Account aanmaken

Ga naar console.firebase.google.com en log in met een Google-account.

2

Project aanmaken

Klik "Add project", kies een naam (bv. popup-restaurant), Google Analytics is optioneel (kan uit).

3

Realtime Database aanmaken

In het linkermenu: Build → Realtime Database → Create database. Kies een regio (Europe) en kies Test mode. Je krijgt nu een URL te zien zoals:

https://jouw-project-default-rtdb.europe-west1.firebasedatabase.app

Kopieer deze URL — die heb je in stap 3 nodig.

⚠️

Test mode: deze stelling is 30 dagen actief. Voor productie pas je later de regels aan, voor tests is dit prima.

2. App online zetten

De app wordt gehost op GitHub Pages — gratis en eenvoudig.

1

GitHub repository aanmaken

Maak een gratis account op github.com, klik New repository en geef het een naam (bv. bestellapp).

2

Bestanden uploaden

Voeg deze bestanden toe aan de root van de repository:

  • index.html — de bediening-app
  • admin.html — het admin dashboard
  • station.html — keuken/bar display
  • help.html — deze handleiding
  • manifest.json + sw.js — voor PWA-installatie
  • icon-192.png + icon-512.png — app-icoontjes

De makkelijkste manier: open de repo op GitHub en typ . (een punt). Dan opent een volledige editor waar je alle bestanden tegelijk kan toevoegen.

3

GitHub Pages activeren

Ga naar Settings → Pages → Source. Kies Deploy from a branch → branch main → folder / (root) → Save.

Na ongeveer 1 minuut is de app beschikbaar op:

https://jouwgebruikersnaam.github.io/bestellapp/

3. Eerste configuratie

Open de app eenmalig op een toestel om alles in te stellen. Daarna nemen alle andere toestellen de configuratie automatisch over.

1

Open het admin dashboard

Ga naar https://jouwgebruikersnaam.github.io/bestellapp/admin.html in een desktop browser.

2

Firebase URL instellen

Ga naar het tabblad Instellingen, plak je Firebase URL en klik Opslaan en verbinden. Het rondje rechtsonder zou groen moeten worden.

3

Configuratie invullen

Vul de basisinstellingen in (deze synchroniseren automatisch met alle andere toestellen):

  • Printers — naam + IP-adres per ticket-printer (later aanpasbaar)
  • Categorieën — bv. Bar, Keuken, Koude keuken — koppel elk aan één of meer printers
  • Menu items — alle gerechten en dranken met prijs en categorie
  • Tafels — beschikbare tafelnummers
  • Medewerkers — naam + PIN-code per persoon
💡

Tip: je kunt het menu en categorieën ook importeren vanuit Excel. Dit is handig om al je items vooraf in een spreadsheet voor te bereiden.

4. Installeren op telefoon

De bediening-app werkt direct in een browser, maar als PWA voelt het als een echte app aan.

1

Open de app

Ga naar https://jouwgebruikersnaam.github.io/bestellapp/ in Safari (iPhone) of Chrome (Android).

2

Eerste keer: log in als admin

Vul je naam in, type PIN 9999, ga naar Firebase synchronisatie en plak de URL. Klik Opslaan en verbinden. Daarna verschijnen alle gedeelde instellingen automatisch.

3

Toevoegen aan startscherm

  • iPhone: deel-icoontje (vierkantje met pijl) → "Zet op beginscherm"
  • Android: drie puntjes rechtsboven → "App installeren"

Rollen en URLs

Verschillende toestellen hebben verschillende rollen. Hieronder een overzicht:

📱 Bediening (smartphone)

Voor wie bestellingen opneemt aan tafel.

URL: /index.html — PIN: 1234 (default)

💻 Admin (laptop)

Voor de beheerder: live overzicht, betalingen, statistieken, instellingen.

URL: /admin.html

📲 Keuken/Bar display (tablet)

Toont openstaande bonnen voor een specifiek station, items afvinken als geserveerd.

URL: /station.html

Testen vóór een event

Doe minimaal één keer een testrun met meerdere toestellen om alles te valideren:

  1. Open de app op 2-3 telefoons en log in met verschillende namen
  2. Plaats elk een paar bestellingen op verschillende tafels
  3. Check of de bestellingen op het admin dashboard verschijnen binnen 5 seconden
  4. Test op de tablet of de juiste categorieën verschijnen
  5. Vink items af op het station-display, check of ze in admin als "geserveerd" verschijnen
  6. Markeer een bestelling als betaald en exporteer naar Excel
  7. Test (indien beschikbaar) ook de printserver met echte printers

Problemen oplossen

Toestel toont geen bestellingen

  • Check of de Firebase URL correct ingegeven is in het Admin/Instellingen tabblad
  • Plak de URL met /test.json erachter in een browser — als null verschijnt, is de URL juist
  • Check de syncstatus rechts: groen rondje = OK, rood = fout, grijs = niet verbonden

Bestellingen worden niet opgeslagen

  • Open F12 → Console en kijk of er fouten staan
  • Check of state.fbUrl niet leeg is — typ dat in de console
  • Controleer dat Firebase nog in Test mode staat (regels niet vervallen)

App werkt offline niet

De app moet eerst online geladen zijn om de service worker te installeren. Bij het volgende bezoek werkt offline-fallback. Bestellingen kunnen offline niet gesynchroniseerd worden — bediening heeft altijd internet nodig.

PIN werkt niet

Check in het admin dashboard of de medewerker (en zijn PIN) in de lijst staat. Default PINs zijn 1234 (bediening) en 9999 (admin) — wijzig deze niet zonder een nieuwe admin-PIN aan te maken.

Functionaliteiten

Volledig overzicht van wat de app kan

📱 Bediening-app

De mobiele app is geoptimaliseerd voor smartphones en gebruikt door bediening om bestellingen op te nemen aan tafel.

Bestellingen opnemen

🔢Tafel + naam combinatie

Elke bestelling is gekoppeld aan een tafelnummer (uit dropdown) en een klantnaam. Hierdoor kunnen meerdere groepen aan dezelfde tafel zitten.

🍽️Items met variaties

Items kunnen variaties hebben (bv. Saignant / À point / Bien cuit voor steak). De juiste variatie wordt op het keukenticket afgedrukt.

💬Commentaar per item

Bij elke orderlijn kan een vrije commentaar toegevoegd worden (bv. "zonder ui", "allergie noten"). Klik op de tekstballon naast de prijs om het te bewerken.

🔀Splitsen / aanvullen

Iemand voegt zich bij een bestaande tafel? Tik op "Splitsen" en voeg de nieuwe naam toe. Items aan een bestaande bestelling toevoegen kan ook via "Toevoegen".

Lopende bestellingen

📋Overzicht actieve tafels

Alle openstaande tafels in één lijst, met live syncstatus. Per tafel zie je naam, totaalbedrag, aantal opnemers en status (open / gedeeltelijk geserveerd / volledig geserveerd).

✓Items markeren als geserveerd

Bij het detail van een bestelling kan je items individueel afvinken. Handig voor wie geen aparte tablet in de keuken heeft.

💶Rekening genereren

De rekening wordt automatisch samengesteld met alle items van alle rondes. Verstuurbaar naar de kassaprinter en/of via de browser afdrukbaar.

Historiek

Apart tabblad met alle afgeronde bestellingen. Toont totale omzet van het event, aantal tafels, en details per bestelling. Rekeningen kunnen herafgedrukt worden.

💻 Admin dashboard

Het dashboard is geoptimaliseerd voor laptops en grote schermen, met meer overzicht en uitgebreide beheerfuncties.

Bestellingen-tab

📊Live KPI's

Bovenaan altijd zichtbaar: open tafels, items in wachtrij, items geserveerd, totale omzet — realtime ververst.

↔️Split-view interface

Links de lijst met actieve tafels (met filtertabs voor open/geserveerd/alle), rechts het detailpaneel met alle items. Items individueel afvinken werkt door direct op de checkbox te tikken.

Statistieken-tab

⏰Activiteit per uur

Staafdiagram per uur met aantal verkochte items en omzet. Toont in één oogopslag de drukke momenten.

🥇Top items en categorieën

Twee tabellen: per item (gesorteerd op aantal verkocht) en per categorie (met percentage van het totaal). Ideaal om na het event te zien wat populair was.

📥Excel export

Eén klik genereert een werkboek met 6 sheets: Per item, Per categorie, Per uur, Uur × Categorie, Per tafel, Alle bestellingen. Volledig detail met datum, tijd, weekdag, opnemer, commentaren.

Historiek-tab

Alle betaalde bestellingen met klap-uit-detail. Per tafel zie je tijdstip van openen en betalen, en de duur. Mogelijk om bestellingen terug te zetten naar status "Open" of "Geserveerd" als er per ongeluk gemarkeerd werd.

📲 Keuken/bar display

Speciale weergave voor een tablet in de keuken of aan de bar. Toont alleen bonnen relevant voor dat station.

🎯Categorie-filter

Bij eerste opstart kies je welke categorieën dit station bedient (bv. enkel "Keuken" of "Bar + Koude keuken"). Bovenaan tabs om te wisselen tussen categorieën met live aantal openstaande items.

⏱️Tijd verstreken per bon

Per bon zichtbaar hoe lang die in de wachtrij staat. Kleurcode: wit (recent), oranje (>8 min), rood (>15 min) — visueel signaal om prioriteit te geven.

✓Per item of volledige bon

Items kunnen één voor één afgevinkt worden door op de checkbox te tikken, of in één keer met de knop "Bon volledig klaar".

⏳Fade-out na 30 seconden

Wanneer alle items van een bon klaar zijn, blijft de bon nog 30 seconden zichtbaar in groen. Tijdens die tijd kan je nog op "Ongedaan" klikken als je per ongeluk afvinkte.

🖨️ Printserver

Optionele Node.js script dat draait op een laptop en automatisch tickets afdrukt op netwerk-printers.

  • Polt elke 2 seconden Firebase voor nieuwe bestellingen
  • Groepeert items per printer op basis van categorie-mapping
  • Stuurt ESC/POS commando's via TCP — werkt met de meeste thermische printers (Epson, Star, Xprinter)
  • Print waiter-naam, tafel, items, variaties en commentaren
  • Configureerbaar: Firebase URL en poll-interval

🔄 Synchronisatie

⚡Realtime tussen toestellen

Alle bestellingen, instellingen, menu, categorieën, printers, medewerkers en tafels worden gedeeld via Firebase. Polling elke 3-5 seconden. Wijzigingen op één toestel zijn binnen enkele seconden zichtbaar op alle andere.

💾Lokale fallback

Elk toestel houdt ook een lokale kopie van alle data in localStorage. Bij tijdelijke verbindingsproblemen blijft de app werken.

🔒Veiligheidscheck

Toestellen pushen pas configuratie naar Firebase nadat ze minstens één keer succesvol gesynchroniseerd hebben. Dit voorkomt dat een nieuw toestel met lege defaults bestaande config zou overschrijven.
